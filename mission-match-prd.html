1. Executive Summary

The Headline

Mission Match enables instant, privacy-first collaboration assessment by having your AI meet their AI — a "digital twin handshake" where agents negotiate consent and exchange profiles to surface high-confidence opportunities.

Problem

Networking events are chaotic and inefficient:

  • Signal/noise problem: Hard to assess technical depth, values alignment, and collaboration fit in quick conversations
  • Trust barrier: People hesitate to share personal info or scan random QR codes from strangers
  • Lock-in anxiety: Talking to one person means missing conversations with others
  • Manual friction: Business cards, LinkedIn requests, and follow-up messages have high drop-off

Current solution (Mission Match v1.0): QR-scan → copy prompt → AI analyzes fit. But it still requires manual data entry by users filling out questions.

Solution

Digital Twin Handshake: Your AI already knows you from your conversation history. When you meet someone:

  1. Scan their QR code
  2. See consent screen showing what will be shared
  3. Your agents exchange structured profiles (digital twins)
  4. Both receive instant compatibility analysis
  5. Get your own shareable profile for future networking

Key innovation: Trust flows through the agent, not direct human-to-human data sharing. Your AI mediates consent on your behalf.

Success

Hackathon Demo Success (March 9):

  • ✅ Working end-to-end demo: 2 people scan → consent → results in <60 seconds
  • ✅ Clear visual showing agent-mediated consent (not direct data share)
  • ✅ Real compatibility analysis from Claude API with actionable opportunities
  • ✅ Judges understand the "digital twin" concept and infrastructure vision

Post-Hackathon Metrics (if built as real product):

  • 50+ active users within first month
  • 80%+ consent completion rate (people approve after seeing consent screen)
  • 4+ average collaboration matches per user

Investment

  • Infrastructure: Supabase for profiles, Claude API for matching

6. User Stories & Jobs to Be Done

User Story US-01: Digital Twin Profile Creation

As a frequent networker
I want my AI to automatically create a collaboration profile from my conversation history
So that I don't have to manually fill out forms at every event

Acceptance Criteria:

  • User can trigger profile creation from Claude interface
  • System extracts: role, mission, proof points, looking-for from recent conversations
  • User can review and edit auto-populated profile before finalizing
  • Profile is stored and reusable across future handshakes
  • Profile includes consent preferences (what data to share)

Priority: Must Have

User Story US-02: Agent-Mediated Consent Flow

As a privacy-conscious professional
I want to see exactly what data will be shared before approving
So that I can trust the handshake process and maintain control

Acceptance Criteria:

  • Consent screen shows: recipient name, data fields being shared, purpose
  • User can select which fields to share (e.g., share role + mission but not LinkedIn)
  • Visual clearly shows "Your AI → Their AI" (not direct human share)
  • User can deny consent without awkwardness (soft "not now" option)
  • Consent is time-limited (expires in 24 hours unless extended)

Priority: Must Have

9. Technical Approach & Constraints

9.1 Proposed Technical Approach (Two Options)

We'll explore both options in parallel during the first week, then commit to one for final implementation.

Option A: Simple API + Claude API (Pragmatic Path)

Architecture:

Mobile Web UI (React/Next.js)
    ↓
Backend API (Node.js + Supabase)
    ↓
Claude API (profile extraction + matching)
    ↓
Supabase Database (profiles, handshakes, analyses)

Tech Stack:

  • Frontend: Next.js (mobile-first), Tailwind CSS (neo-brutalist design from v1.0)
  • Backend: Next.js API routes (serverless functions)
  • Database: Supabase (PostgreSQL)
  • AI: Claude API (Sonnet 3.5 for speed, Opus 4.5 if need deeper analysis)
  • QR Codes: QRServer API or qrcode.js library
  • Hosting: Vercel (free tier for hackathon)

Pros:

  • ✅ Fast to build (can ship in 1 week)
  • ✅ Works with any AI assistant (not just Claude)
  • ✅ Hosted infrastructure (no local setup required)
  • ✅ Easy to demo on stage (just show QR code)

Cons:

  • ❌ Not leveraging Anthropic-specific tech (MCP)
  • ❌ Less novel from infrastructure perspective
  • ❌ Requires internet connection for handshakes

Option B: MCP Extension (Novel Path)

Architecture:

Local MCP Server (Your Machine)
    ↓
Claude Desktop App
    ↓
Handshake Coordination API (Cloud)
    ↓
Remote MCP Server (Their Machine)

Pros:

  • ✅ Novel use of MCP (could influence Anthropic's roadmap)
  • ✅ Positions as infrastructure play (not just app)
  • ✅ Local data control (privacy-first)
  • ✅ Shows deep technical chops to judges

Cons:

  • ❌ Requires both users running MCP servers locally (not realistic for demo)
  • ❌ Complex setup for hackathon timeline
  • ❌ Risk: May not finish in time for March 9

Glossary

  • Digital Twin: Structured profile representing a person, auto-generated from their AI conversation history
  • Handshake: The bilateral consent flow where two people agree to exchange profiles
  • Agent-Mediated Consent: Trust model where AI agents negotiate data sharing on behalf of users (not direct human-to-human)
  • Heartware: Positioning concept - "API for human context" or "OAuth for AI-to-AI consent"
  • MCP (Model Context Protocol): Anthropic's open protocol for Claude to connect to external data sources
  • Track 2: Hackathon track "AI Companions with Purpose" - AI uses personal data to provide value to user
