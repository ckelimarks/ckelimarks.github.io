What is Claude Code?

Claude Code is a tool from Anthropic that lets you work with Claude directly in your terminal. Unlike the chat interface, Claude Code can read and write real files on your computer, run commands, and build things with you — not just talk at you.

Your Personal OS lives in actual folders and markdown files that you own. Not locked inside someone else's app. Local, private, yours.

What you need

  • A Claude Pro or Max subscription (the free plan doesn't include Claude Code)
  • A computer — Mac, Windows, or Linux all work

That's it — no other dependencies. There's also a Desktop app if you'd prefer a graphical interface over the terminal.

Opening your terminal (the hardest part, honestly)

This is the step that trips people up, but it's simpler than it sounds. You just need to open the app on your computer that lets you type commands:

  • Mac: Open Terminal (search for it in Spotlight with ⌘+Space) or use iTerm2 if you have it
  • Windows: Open PowerShell (search for it in the Start menu) — or if you have Git for Windows, Git Bash works great too
  • Linux: Open your terminal app — Bash, Zsh, whatever you normally use

You'll see a window with a blinking cursor. That's where you type the commands below.

Install Claude Code

Copy-paste this into your terminal and hit Enter:

# Mac / Linux
curl -fsSL https://claude.ai/install.sh | bash

# Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex

It'll ask you to log in with your Claude account in your browser. Follow the prompts — takes about a minute.

Start your Personal OS

Once installed, create a folder for your system and launch Claude Code inside it:

mkdir my-personal-os && cd my-personal-os
claude

Paste your chosen prompt, and you're off.

You don't need to read this before using the prompt — it's here for people who want to understand the architecture.

CLAUDE.md — The persistent brain

A markdown file at the project root that Claude reads every time you start a session. Holds your profile, active projects, session history, and compaction seeds (compressed state summaries that let a new session reconstruct context fast). This is how Claude "remembers" across sessions without any external database.

Compaction seeds

When your context gets long, Claude compresses the essential state into dense summaries — project statuses, active decisions, unresolved questions, and things you explicitly said to remember. These seeds live in CLAUDE.md and get updated at the end of every session, so nothing critical is lost even as conversations roll forward.

agents.md — Subagent definitions

A file that defines specialized agents you can invoke for specific tasks — a Daily Ops agent for morning check-ins, a Project Context agent for switching between work, a Review agent for weekly summaries. Each definition includes what files it reads/writes, what triggers it, and any constraints. Think of it as a team of specialists you can call on.

Slash commands

Custom commands you type in the terminal — like /morning to start your day, /switch projectname to change context, or /end-day to wrap up. These automate the rituals and workflows you described in the interview so they're one keystroke away.

Hooks

Automated triggers that fire at specific moments — when you start a session (load context), before a commit (validate memory is current), after a commit (update the session log). These handle the housekeeping so you don't have to think about it.

What it uses

  • Markdown — Plain text files you can read anywhere, forever. No vendor lock-in.
  • Git — Version control so every change is tracked. You can always go back.
  • Local files — Everything on your machine. Nothing sent anywhere unless you choose.
  • Claude Code — The engine that reads, writes, and orchestrates everything.

How to evolve it

Your system is just files. Edit them in any text editor. Ask Claude to add features or restructure things. Share your setup with others. Back it up however you back up anything else. You own it — it's not a subscription, it's not an app that might shut down.

What it won't do — let's be straight

  • It doesn't run in the background. Claude is active when you're using it.
  • It doesn't sync across devices automatically (unless you add a Git remote).
  • It doesn't replace your calendar, email, or task app. It's a thinking layer alongside them.
  • It's files on your computer. You own them completely — but you maintain them too.
